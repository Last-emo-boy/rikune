Get Started开始使用

Choose the deployment shape that matches your risk model: static-only Linux Docker, Windows-native auto-sandbox, or split analyzer/runtime. 根据你的风险模型选择部署形态：纯静态 Linux Docker、Windows 原生 auto-sandbox，或拆分式 analyzer/runtime。

Best-practice runtime split: use docker-compose.analyzer.yml for static-only Linux analysis with RUNTIME_MODE=disabled, switch to manual or remote-sandbox only when you intentionally connect a Windows Runtime Node / Host Agent, and reserve auto-sandbox for Windows-native analyzers. Docker or WSL analyzers cannot use auto-sandbox directly. 推荐实践：使用 docker-compose.analyzer.yml 和 RUNTIME_MODE=disabled 进行纯静态 Linux 分析；只有在你明确接入 Windows Runtime Node / Host Agent 时，才切换到 manual 或 remote-sandbox；auto-sandbox 仅适用于 Windows 原生 analyzer。Docker 或 WSL 中的 analyzer 不能直接使用 auto-sandbox。
